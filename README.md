# Restaurant & Product Explorer System

# Overview

This project is a Python task. It acts like a restaurant management system. The system also uses real-world API handling with error logging.

The system shows how to use data structures in a way. It also shows file handling, API integration and strong error handling. This is like how real applications work.

# Part 2: Restaurant Management System

# Task 1: Menu Exploration

* The menu is shown grouped by category.

* We find out which items are available and which are not.

* We calculate these things:

* items

* Available items

* The expensive item

* Items that cost less than ₹150

# Task 2: Cart Operations

* A cart system is made using a list of dictionaries.

* The cart can do these things:

* Add an item (with checks)

* Update the quantity of an item (no duplicates allowed)

* Remove an item

* A final order summary is made with GST calculation.

# Task 3: Inventory Tracker

* We use `copy.deepcopy()` to make a backup.

* We act like an order is fulfilled.

* We make sure there is no stock.

* We make alerts for items that're low in stock.

#Task 4: Sales Log Analysis

* We calculate how much money is made per day.

* We find the selling day.

* We find the ordered item.

* We add a day and update the results.

* We print all orders using `enumerate()`.

# Part 3: Product Explorer & Logger

### 🔹 Task 1: File Handling

* A file named `python_notes.txt` is made.

* More content is added to the file.

* The lines in the file are. Shown with numbers.

* A search for keywords is made (it doesn't matter if its lower case).

# Task 2: API Integration

* Product data is taken from DummyJSON API.

* A table of products is shown in a format.

* Products with ratings are filtered.

* Products are sorted by price.

* Products of a category are taken.

* A POST request is acted like it is sent.

# Task 3: Error Logging

* A custom logger for errors is made.

* Errors are logged with timestamps.

* These are handled:

* Errors with connections

* HTTP errors (like 404)

* Logs are stored in `error_log.txt`.

# How to Run

1. Open the project folder.

2. Run these Python files:

```
python part2_restaurant.py

python part3_file_api_logger.py

```
